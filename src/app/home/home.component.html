The XLE+Glue web application

This page presents several resources used for exploring LFG's glue semantics. Concretely, it presents the Glue semantics workbench and the LiGER rewrite system . It also serves as a revised frontend for the original XLE + Glue. As all of these resources, this frontend is open source. It is available at: XLE+Glue client

Quick start

Running the system locally

To run this system, you require:

  • A running version of the GSWB
  • A running version of LiGER
  • A running version of the client

You either need to compile your own jar files for the GSWB and LiGER, or you can use the ones provided at XLE+Glue resources. Please contact Mark-Matthias Zymla if you can't access the relevant files. You need to run both -jar files with the -web parameter. Furthermore, they require resource folders to be present in the same directory as the jar files. These resource folders contain additional requirements explained on the corresponding github pages.

The frontend is written in angular. To use it locally, you need to install node.js and npm. Then, you can run ng build in the client directory to build the frontend, which is then stored in the dist directory. You can use http-server to serve the frontend. It can be installed with npm install -g http-server. Then, you can run http-server dist/xleplusglue-cient to serve the frontend. Http-server will tell you the correct address.

If you have node.js and http-server installed, you can also try to use the pre-build version available at: . There, you also find the relevant resource folders for the GSWB and LiGER and a .sh file that allows you to execute the backend (i.e., the GSWB and LiGER jars) and frontend with a single command.
